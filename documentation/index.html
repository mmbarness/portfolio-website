Prerequisites

Before you begin, make sure your development environment includes Node.js® and an npm package manager.

Node.js

React requires Node.js version >= 8.10 on your local development machine (but it’s not required on the server).

  • To check your version, run node -v in a terminal/console window.

  • To get Node.js, go to nodejs.org.

Then you need to install all the dependencies. To do this, in the terminal / console window, run npm install

All commands run in the project folder!

Development server

Run npm start for a dev server. Navigate to http://localhost:3000/. The app will automatically reload if you change any of the source files.

Build

Run npm run build to build the project. The build artifacts will be stored in the build/ directory.

After that, your application will be ready for upload to the server. If you have any problems you can read this article React App Deployment.

File structure


package.json

All npm packages used in the application


src

Application source


public

Static files such as images or json data

Static files

data

Data used in the application.

These files store user information. If you need to make any changes, there is no need to rebuild the project, you just need to add this information to the corresponding file.

  • experience.json - for experience list
  • person.json - for person information
  • projects.json - for project list
  • trust.json - for testimonial list, companies list, skills and education

img

Images used in the application

Styles

Styles in scss format. This folder contains styles for the entire application.

Change color scheme

The easiest way to change the color scheme is to change the values of variables in the vars.scss file.
For example, the variable $h1-font-size sets the size of the H1 headers.
More specific styles are contained in components.

Components

components folder contains all components used in the application. They are divided into two groups, components used for pages and reusable(shared) components.


app.js

Main application file. Routing rules are defined in this file.


shared

Reusable components

  • contact-form - the component that creates the contact form
  • footer - the component that creates the application footer
  • navbar - the component that creates the application navbar

pages components

  • about - component responsible for the about page.
  • contact - component responsible for the contact page.
  • home - component responsible for the home page.
  • no-match - component responsible for the 404 page.
  • projects - component responsible for the projects page.
  • single-project - component responsible for the single project page.

You can edit pre-made page components or create new ones.

Create new page

To create a new page, you need to create a new component for this page. Just create new file in components folder.
For example components/landing-page/landing-page.js 

        import React from 'react'

        const LandingPage = () => {
          return (
            <h1>Landing Page</h1>
          )
        }

        export default LandingPage

Now you need to create a new route for the page. Open file app.js and import component
import {LandingPageComponent} from 'import LandingPage from './pages/landing-page/landing-page'
then insert string <Route path="/landing-page" component={LandingPage}/>,
before <Route component={NoMatch}/>

Great, now you have a new page and you can see it http://localhost:3000/landing-page or your_domain/landing-page

For more information please see React and Create React App
